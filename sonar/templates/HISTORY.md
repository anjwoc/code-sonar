|:---|:---|

## 4. 분석 시 주의점

- 코드 변경과 문서 변경이 어긋나는 항목을 기록한다.
- 삭제된 기능은 실제 호출부가 남아 있는지 확인한다.
- 이벤트/배치 변경은 후속 consumer와 저장소 영향까지 확인한다.

## 수집 명령 예시

```bash
git log --format="%h | %ai | %s" --stat -20
git diff --stat {base}..{head}
git log --oneline -- {관심 파일 또는 디렉터리}
```
